# ターミナル使い分けガイド

## 方針

|      | WezTerm                                             | Ghostty                                  |
| ---- | --------------------------------------------------- | ---------------------------------------- |
| 役割 | 日常開発（メイン）                                  | SSH接続・軽量作業（サブ）                |
| 強み | Lua設定、Neovim連携、タブ/ペイン/ワークスペース管理 | 軽量・高速描画、最小限の設定             |
| 用途 | コーディング、Claude Code、ローカル作業全般         | リモートサーバー接続、ログ監視、簡易作業 |

## 背景

ryoppippiさんの開発スタイルに基づく判断:

- WezTermの設定がLuaで書けるため、Neovimの設定スキルと相互に活きる
- tmuxを使わず、WezTerm自体のタブ/ペイン/ワークスペース機能で完結できる
- WezTerm CLIからNeovim連携が可能
- SSH接続時はターミナル側の高度な機能を使わないため、Ghosttyの軽さが活きる
- ターミナルを見分けることで「これはリモート作業」と視覚的に区別できる
